## 根拠説明
本提案は、「Hard Fork to Protocol Version 11 ('van Rossem' Hard Fork)」に関するHard Fork Initiation Governance Actionです。提案本文では、CardanoメインネットをProtocol Version 11へアップグレードすること、新たなPlutusプリミティブの導入、Plutus V1・V2・V3間の組み込み関数の統一、組み込み型に対するcase式のサポート、ならびに台帳ルールおよびノードレベル機能の改善について説明されています。
Cardano憲法第2条第6節第1項および第2項について、本提案はIPFSを利用した不変のオフチェーン参照を採用しています。さらに、提案本文にはsummary、motivation、rationale、rollback plan、およびreferencesが含まれていることから、Governance Actionとして評価するために十分な情報が提示されていると判断します。
Cardano憲法第2条第6節第3項について、提案本文にはIntersect Hard Fork Working Groupによる推奨、Technical Steering Committeeによる承認に加え、テスト結果、セキュリティ監査、および性能評価結果が含まれています。これらに基づき、Cardano憲法が求める十分な技術的レビューおよび精査プロセスが実施されたと判断します。
さらに、私たちは本提案がHARDFORK-01、HARDFORK-02a、およびHARDFORK-03に定められたプロトコルバージョン変更要件を満たしていることを確認しました。
また、本提案では新たなプロトコルパラメータや新たなPlutusバージョンは導入されず、HARDFORK-05からHARDFORK-08との明確な抵触も確認されませんでした。
HARDFORK-04aでは、アクティブステークベースで少なくとも85%のステークプールが新しいプロトコルバージョンに対応したノードへアップグレードしていることが望ましいとされています。85%達成については、私たちはアクティブステーク基準で85%以上に到達していることを確認できませんでしたが、このガードレールは必須要件である「must」ではなく「should」として規定されています。また、PoolTool、Cexplorer、およびIntersect Hard Fork Working Groupのreadiness trackerを含む複数の公開指標では、おおむね80〜86%程度のアップグレード率が示されていました。これらを総合的に考慮した結果、本提案を違憲と判断するほどの重大な懸念は認められませんでした。
したがって、私たちは本提案を合憲であると判断します。
